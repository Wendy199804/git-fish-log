# 🐟 fish - Git 摸鱼 & 爆肝分析器

`fish` 是一个好玩有趣的命令行工具（CLI），专为程序员设计。它通过扫描本地 Git 仓库的 Commit 历史（支持多项目扫描和自动发现），深度挖掘你的开发轨迹，自动计算出你每天的**“摸鱼指数”**，生成幽默风趣的周报与月报，并配有令人会心一笑的 **AI 风格无情吐槽**！

---

## ✨ 核心特色

1. **摸鱼指数算法**：基于每日提交次数与提交时间跨度进行轻量级多维拟合。0 次提交为 99% 摸鱼，频繁提交且跨度长达 8 小时以上为 10% 狂热爆肝。
2. **多维度报告展示**：
   - **默认周报**：按天梳理工作轨迹、努力/轻松天数以及趣味 锐评。
   - **月度分析 (`-m`)**：利用常规 Git 提交前缀正则分类（`feat`/`fix`/`chore`/`docs`），生成图形化比例条与深度吐槽。
   - **项目排行榜 (`-p`)**：对比你名下各个项目的爆肝程度，定位你的“主力搬砖地”与“划水摸鱼点”。
   - **黄金时间段分析 (`-t`)**：绘制 24 小时 commit 频次分布柱状图，精准识别你的下班突击时间和开会摸鱼期。
   - **深夜幽灵检测 (`-g`)**：统计半夜 12 点至凌晨 5 点的 commits，劝阻无谓爆肝，保命要紧！
3. **多仓库/多项目管理**：支持将多个 Git 仓库注册到全局配置中，甚至支持一键递归扫描某目录下所有的 Git 仓库。
4. **极速零阻塞体验**：纯原生 `git log` 命令执行并返回结构化数据，不依赖大型第三方封装，由 `tsup` 预打包为极轻量独立 ESM 运行。

## 🚀 安装与使用指南

### 1. 克隆与安装依赖
```bash
npm install
```

### 2. 编译打包
```bash
npm run build
```
编译产物会生成在 `./dist/index.js`

### 3. 全局链接 CLI 命令
在当前目录中运行：
```bash
npm link
```
这将在你全局的 `npm` bin 目录中创建符号链接。你现在可以在系统的任何目录中直接通过 `fish` 运行该程序了！
### 4.配置【划重点】
配置/新增 GitLab 远程扫描源（支持多数据源管理）：
  ```bash
  fish config gitlab <token> [host] [name]
  ```
  * `token`: 你的 GitLab 个人/项目访问令牌 (PAT)，需有 `read_api` 或 `api` 权限。
  * `host`: 可选。私有部署的 GitLab 域名（如 `https://gitlab.mycompany.com`），缺省默认使用 `https://gitlab.com`。
  * `name`: 可选。给该数据源指定的别名（如 `my-company`），缺省时自动使用 host 域名作为别名。若别名已存在则会覆盖更新。

## 🎨 算法说明：摸鱼指数计算公式（v2 重设计）

### 设计哲学

摸鱼指数（0~100）是反向指标——**越高越摸鱼**。算法的核心假设是：

> Commit 次数、持续工作时长、提交分布，比代码行数更能体现"没摸鱼"。
> 代码行数仅作辅助参考，使用对数压缩并降低权重。

### 指标区间直觉

| 摸鱼指数 | 含义 | 典型场景 |
|----------|------|----------|
| 95~100 | 几乎没干活 | 0 commit，或仅 1~2 行微调 |
| 70~85 | 小需求、小修复，整体悠闲 | 1~3 个有效 Commit，1~3 小时 |
| 40~60 | 正常搬砖 | 4~7 个有效 Commit，4~8 小时 |
| 20~40 | 爆肝工作 | 8~12 个有效 Commit，8~12 小时 |
| 0~20 | 通宵上线、人形 CI/CD | 13+ 有效 Commit，12+ 小时 + 深夜 |

### 核心概念：有效 Commit

只有单次修改行数（additions + deletions）**≥ 20 行**的 Commit 才算"有效 Commit"。
这可以天然过滤掉：

- 格式化 / 缩进调整
- 单行 typo 修复
- 配置微调
- 注释增删

用 \(N_{eff}\) 表示有效 Commit 数量，只有它参与核心计分。

### 五项子指标

每天同时计算 **五个维度**的指数：

| 指数 | 说明 |
|------|------|
| 🐟 摸鱼指数 | 反向指标，越高越摸鱼 |
| 🌙 修仙指数 | 深夜 (00:00-05:00) 提交频率 |
| 🧱 搬砖指数 | 代码量 + 有效提交频率 |
| 💥 爆发指数 | 单次提交修改量（一把梭） |
| 🏷 人格标签 | 任意指数 ≥ 80 时触发 + 隐藏成就 |

### 输入数据

- \(N\) = 当天 Commit 总数
- \(N_{eff}\) = 有效 Commit 数（增删 ≥ 20 行）
- \(S\) = 最早与最晚 Commit 时间跨度（小时）
- \(L\) = 总修改行数 (additions + deletions)
- \(H_{active}\) = 有至少 1 次 Commit 的小时数
- \(Night\) = 深夜 (00:00-05:00) 提交数
- \(Weekend\) = 是否为周末

### 子分数计算（各项 0 ~ 100）

#### I₁ 有效提交分（权重 35%）

\[
commitScore = \min(100,\ \log_2(N_{eff} + 1) \times 20)
\]

对数衰减防刷 Commit。校准点：

| \(N_{eff}\) | 0 | 1 | 2 | 3 | 5 | 7 | 10 | 15 |
|--------------|---|---|---|---|---|---|----|-----|
| commitScore | 0 | 20 | 32 | 40 | 52 | 60 | 69 | 80 |

#### I₂ 工时跨度分（权重 30%）

\[
spanScore = \min(100,\ \sqrt{S} \times 22)
\]

平方根防无限拉长跨度。校准点：

| \(S\) (h) | 0.5 | 1 | 2 | 4 | 6 | 8 | 10 | 12 |
|------------|-----|----|----|----|----|----|----|-----|
| spanScore | 16 | 22 | 31 | 44 | 54 | 62 | 70 | 76 |

#### I₃ 时间分布分（权重 20%）

\[
spreadScore = \min(100,\ H_{active} \times 12)
\]

有 Commit 的小时数越多，说明不是集中突击、而是持续投入。

| \(H_{active}\) | 1 | 2 | 3 | 4 | 5 | 6 | 7 | 8+ |
|-----------------|---|---|---|---|---|---|---|-----|
| spreadScore | 12 | 24 | 36 | 48 | 60 | 72 | 84 | 96+ |

#### I₄ 代码行数加分（权重 10%）

\[
lineBonus = \min(100,\ \log_{10}(L + 1) \times 25)
\]

log₁₀ 重度压缩 + 10% 低权重，避免 200 行被误判为大量工作。

| \(L\) | 10 | 100 | 200 | 500 | 1000 | 2000 | 5000 |
|--------|-----|------|------|------|-------|-------|-------|
| lineBonus | 26 | 50 | 58 | 68 | 75 | 83 | 93 |

#### I₅ 深夜加成（+5 分）

\[
nightBonus = Night > 0\ ?\ 5 : 0
\]

有深夜提交直接加 5 分工作分，不参与权重乘法。

### 最终公式

\[
\begin{aligned}
workScore &= commitScore \times 0.35 \\
           &+ spanScore \times 0.30 \\
           &+ spreadScore \times 0.20 \\
           &+ lineBonus \times 0.10 \\
           &+ nightBonus \\[4pt]
if\ (isWeekend\ \&\&\ N > 0):\quad workScore &+= 5 \\[4pt]
hardworking &= \min(100,\ round(workScore)) \\[4pt]
fish &= \max(1,\ 100 - hardworking)
\end{aligned}
\]

#### 修仙指数

\[
nightScore = \min(100,\ Night \times 30)
\]
\[
nightOwl = Night = 0\ ?\ 0 : \min(100,\ round(nightScore \times 0.7 + spanScore \times 0.3))
\]

#### 搬砖指数

\[
builder = \min(100,\ round(lineBonus \times 0.6 + commitScore \times 0.4))
\]

#### 爆发指数

\[
avgLines = N_{eff} > 0\ ?\ \frac{effLines}{N_{eff}} : \frac{L}{\max(N,1)}
\]
\[
burst = \min(100,\ round(\log_2(avgLines + 1) \times 12))
\]

| 场景 | N | \(N_{eff}\) | S(h) | \(H_{active}\) | L | fish |
|------|---|--------------|------|------------------|---|------|
| 摸鱼一天 | 0 | 0 | 0 | 0 | 0 | **95** |
| 小修小补 | 1 | 0 | 0.1 | 1 | 15 | **89** |
| 小需求 | 2 | 1 | 1 | 1 | 150 | **80** |
| 轻量一天 | 3 | 2 | 3 | 2 | 300 | **69** |

| 较忙 | 8 | 7 | 8 | 5 | 900 | **39** |
| 爆肝 | 12 | 10 | 10 | 7 | 1500 | **28** |
| 通宵上线 | 18 | 15 | 14 | 10 | 3000 | **16** |

### 特殊情况

- **0 Commit**：摸鱼 = 95（不判 100，可能开会/调研/排查问题），其余为 0，标签 = "🐟 今日暂无代码活动"
- **纯刷 Commit（大量 <20 行的提交）**：\(N_{eff}\) = 0 → commitScore = 0，工作分主要来自 span/spread，摸鱼指数保持在 80+
- **单次大提交（如 500 行 1 次 Commit）**：commitScore = 20，spreadScore ≤ 12，工作分 ≈ 17，摸鱼 ≈ 83 — 一把梭不算持续工作
